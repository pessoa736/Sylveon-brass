# Sylveon Brass

> Um pacote de mods para Minecraft que junta fábricas industriais, exploração de mundos novos e a aventura de capturar Pokémon — tudo em um só jogo.

## O que é isso?

**Sylveon Brass** é um pacote customizado para Minecraft (versão 1.21.1) criado para um servidor privado de amigos. A ideia era simples e maluca ao mesmo tempo: misturar três estilos de jogo que normalmente não se encontram.

- **Construir fábricas** com máquinas, engrenagens e esteiras que produzem coisas sozinhas
- **Explorar mundos novos** além do mapa normal, com dungeons e estruturas desafiadoras
- **Capturar e treinar Pokémon** enquanto você minera, constrói e explora

O resultado? Um jogo onde você pode montar uma fábrica automatizada de mineração enquanto seu Pokémon favorito te acompanha na jornada.

## O que tem de especial?

- **Combinações inesperadas**: imagine uma fábrica de processamento de minério com um Pikachu correndo pelo chão
- **Tudo configurado à mão**: cada mod foi ajustado para funcionar bem com os outros, sem quebrar a experiência
- **Pensado para rodar bem**: otimizações para o jogo não travar mesmo com tanta coisa acontecendo ao mesmo tempo
- **Fácil de instalar**: disponível na CurseForge, com um clique você joga
- **Documentação clara**: tem guia tanto pra quem quer só jogar quanto pra quem quer entender como o pacote foi feito

## Como o projeto funciona por trás

O projeto usa uma ferramenta chamada **Packwiz**, que funciona como um "gerenciador de receitas" para pacotes de Minecraft. Em vez de sair caçando mods na internet, tudo fica organizado em arquivos de texto que podem ser versionados (como código de programação).

O fluxo é mais ou menos assim:

1. **Adicionar ou remover mods** editando o arquivo principal
2. **Ajustar configurações** de cada mod para eles funcionarem bem juntos
3. **Gerar o pacote final** em formato compatível com CurseForge ou Modrinth
4. **Publicar** para que qualquer pessoa possa baixar e jogar

## O que aprendi fazendo isso

- **Organizar centenas de peças** que dependem umas das outras para funcionar
- **Resolver conflitos** quando dois mods tentam mudar a mesma coisa
- **Equilibrar a experiência** entre mods com estilos muito diferentes (tecnologia vs. aventura)
- **Cuidar de versões** de um projeto que não é exatamente "código", mas precisa ser organizado como se fosse
- **Escrever documentação** que ajuda outras pessoas a usar e contribuir
